# Interface Brainstorming — Evaluation & Integration

## Evaluation Criteria

Each proposal should strongly optimize for a different **optimization dimension**:

| Proposal | Optimization |
|---|---|
| A | Familiarity |
| B | Conceptual reframing |
| C | Experiential leverage through technology |
| D | Essentialism and reduction |
| E | Expert speed and fluency |

---

## Post-Selection Integration

After the user selects a direction, the chosen interface proposal feeds back into the broader planning workflow:

1. **Shape Up Planning** — the selected direction is reconciled into the shaping proposal (scope, risks, boundaries)

2. **Review Gate** — the complete shaping artifact (including the selected interface direction) is submitted for approval

3. **Tech Planning Sequencing** — after approval, the shaped proposal (with converged UX direction) is passed to tech-planning for implementation sequencing

**This ensures the interface direction is validated within the full strategic context, not in isolation.**
### Post-Selection Integration
The recommendations should **help convergence, not only divergence**.
