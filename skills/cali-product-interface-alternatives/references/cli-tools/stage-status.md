# Stage Status Helper

> Utility to update workflow stage progress for human reviewers.

---

## Commands

| Command | Description |
|---------|-------------|
|| `/pw-setphase stage=N` | Set current stage by index (0=Setup, 1=Context, ...) or `phasename=Name` |
|| `/pw-next` | Advance to next stage |
|| `/pw-status` | Show workflow status |

---

## Stage Index (0-indexed for CLI)

| Index | Stage Name | Workflow Stage |
| 0 | Setup | Stage 1 |
| 1 | Context | Stage 2 |
| 2 | Shape | Stage 3 |
| 3 | Interface | Stage 4 |
| 4 | Critique | Stage 5 |
| 5 | Gate | Stage 6 |
| 6 | Scope Adjustment | Stage 7 |
| 7 | Interface Gate | Stage 8 |
| 8 | Interface Selection | Stage 9 |
| 9 | Tech Planning | Stage 10 |
| 10 | Execution | Stage 11 |
| 11 | Verification | Stage 12 |
| 12 | Execution Critique | Stage 13 |
| 13 | Complete | Workflow Done |

---

## Examples

```bash
# Show current status
/pw-status

# Set to Shape stage
/pw-setphase phasename=Shape

# Advance to next stage
/pw-next
```

```
│   4/15 — shape:20 (in-progress)         │
├─────────────────────────────────────────┤
│   0 triage       ● Done                 │
│   1 select       ● Done                 │
│   2 setup        ● Done                 │
│   3 context      ○ Skipped              │
│   4 shape        ◆ shape:20 (current)   │
│   5 critique     ○ Pending              │
│   6 gate         ○ Pending              │
│   ...                                    │
└─────────────────────────────────────────┘
```

This is generated by the skill at the start of each stage.

---

## Auto-Update vs Manual

> ⚠️ **0-indexed for CLI** — `/pw-setphase stage=2` points to index 2, which is `setup`.
> Prefer phasename (immune to reordering): `/pw-setphase phasename=Shape`.

**Auto-update:** The extension updates `currentStage` via `/pw-next`, `/pw-setphase`.

**Manual update:** If the LLM completes a stage but didn't update:
```bash
/pw-setphase phasename=Shape  # preferred
/pw-setphase stage=2          # 0-indexed: 0=triage, 1=select, 2=setup, 3=context, etc.
```
