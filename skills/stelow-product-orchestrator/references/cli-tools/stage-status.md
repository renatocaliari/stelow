# Stage Status Helper

> Utility to update workflow stage progress for human reviewers.

---

## Commands

| Command | Description |
|---------|-------------|
|| `/sw-setphase stage=N` | Set current stage by index (0=Setup, 1=Context, ...) or `phasename=Name` |
|| `/sw-next` | Advance to next stage |
|| `/sw-status` | Show workflow status |

---

## Stage Index (0-indexed — matches extension PHASE_NAMES)

| Index | Phase | Description |
|-------|-------|-------------|
| 0 | Triage | Inbox Triage — auto if list detected |
| 1 | ItemSelect | Item Selection — after triage |
| 2 | Setup | Project Setup — stage selection, safe-change |
| 3 | Context | Strategic Context (optional) |
| 4 | Shape | Shape Up — spec-product output |
| 5 | Critique | Product Critique |
| 6 | Gate | Review Gate (Plannotator) |
| 7 | Scope | Scope Adjustment |
| 8 | Interface | Interface Alternatives |
| 9 | Int. Gate | Interface Gate (Plannotator) |
| 10 | Selection | Interface Selection |
| 11 | Planning | Tech Planning |
| 12 | Execution | Execution |
| 13 | Verification | Verification |
| 14 | Audit | Execution Critique |

---

## Examples

```bash
# Show current status
/sw-status

# Set to Shape stage
/sw-setphase phasename=Shape

# Advance to next stage
/sw-next
```

## Auto-Advance vs Manual

> ⚠️ **0-indexed for CLI** — `/sw-setphase stage=2` points to index 2, which is `setup` (see table above).
> Prefer phasename (immune to reordering): `/sw-setphase phasename=Shape`.

**Auto-advance (default):** LLM updates `stelow.json` directly via bash and proceeds to next stage without stopping. See SKILL.md Stage Transitions for the bash command.

**Manual (paused/error recovery only):** Use `/sw-next` or `/sw-setphase`:
```bash
/sw-setphase phasename=Shape  # preferred
/sw-setphase stage=2          # 0-indexed: 0=triage, 1=select, 2=setup, 3=context, etc.
```

**Reshape cycle (planning → shape):** When tech planning discovers a blocking constraint (`planning:15`), the workflow resets to Shape via `/sw-setphase phasename=Shape`. The blocking constraint is saved to `context/blocking-constraints.md` and automatically consumed by `shape:10` when the new cycle starts. This reuses the existing `/sw-setphase` mechanism — no new command.
