# Stage Status Helper

> Utility to update workflow stage progress for human reviewers.

---

## Commands

| Command | Description |
|---------|-------------|
|| `/pw-setphase stage=N` | Set current stage (0-indexed: 0=Setup, 1=Context, ...) |
|| `/pw-next` | Advance to next stage |
|| `/pw-status` | Show workflow status |

---

## Stage Index (0-indexed for CLI)

| Index | Stage Name | Workflow Stage |
| Index | Stage Name | Workflow Stage |
| 0 | Setup | Stage 1 |
| 1 | Context | Stage 2 |
| 2 | Shape | Stage 3 |
| 3 | Interface | Stage 4 |
| 4 | Critique | Stage 5 |
| 5 | Gate | Stage 6 |
| 6 | Scope Adjustment | Stage 7 |
| 7 | Interface Gate | Stage 8 |
| 8 | Interface Selection | Stage 9 |
| 9 | Tech Planning | Stage 10 |
| 10 | Execution | Stage 11 |
| 11 | Delivery Audit | Stage 12 |
| 12 | Complete | Workflow Done |

---

## Examples

```bash
# Show current status
/pw-status

# Set to Stage 3 (Shape Up)
/pw-setphase stage=2

# Advance to next stage
/pw-next
```

This is generated by the skill at the start of each stage.

---

## Auto-Update vs Manual

**Auto-update:** The extension updates `currentStage` when user uses `/pw-next`, `/pw-setphase`.

**Manual update:** If the LLM completes a stage but the extension didn't update (e.g., LLM ran the skill standalone), the user can manually update:

```bash
/pw-setphase stage=N
```

Where N is the 0-indexed stage number (2 for Stage 3, 6 for Stage 7, etc.)

## Delivery Audit (Stage 12)

After all scopes are executed, the Delivery Audit stage verifies:

1. **Scope completeness** — all planned scopes implemented
2. **Gap analysis** — identify missing or extra features
3. **Quality gates** — tests passing, lint clean, build succeeds
4. **Lessons learned** — document what went well/what could improve

Outputs `delivery-audit.md` with findings and decision matrix.
