# Stage Status Helper

> Utility to update workflow stage progress for human reviewers.

---

## Commands

| Command | Description |
|---------|-------------|
|| `/pw-setphase stage=N` | Set current stage by index (0=Setup, 1=Context, ...) or `phasename=Name` |
|| `/pw-next` | Advance to next stage |
|| `/pw-status` | Show workflow status |

---

## Stage Index (0-indexed — matches extension PHASE_NAMES)

| Index | Phase | Description |
|-------|-------|-------------|
| 0 | Triage | Inbox Triage — auto if list detected |
| 1 | ItemSelect | Item Selection — after triage |
| 2 | Setup | Project Setup — stage selection, safe-change |
| 3 | Context | Strategic Context (optional) |
| 4 | Shape | Shape Up — spec-product output |
| 5 | Critique | Product Critique |
| 6 | Gate | Review Gate (Plannotator) |
| 7 | Scope | Scope Adjustment |
| 8 | Interface | Interface Alternatives |
| 9 | Int. Gate | Interface Gate (Plannotator) |
| 10 | Selection | Interface Selection |
| 11 | Planning | Tech Planning |
| 12 | Execution | Execution |
| 13 | Verification | Verification |
| 14 | Audit | Execution Critique |

---

## Examples

```bash
# Show current status
/pw-status

# Set to Shape stage
/pw-setphase phasename=Shape

# Advance to next stage
/pw-next
```

## Auto-Update vs Manual

> ⚠️ **0-indexed for CLI** — `/pw-setphase stage=2` points to index 2, which is `setup` (see table above).
> Prefer phasename (immune to reordering): `/pw-setphase phasename=Shape`.

**Auto-update:** The extension updates `currentStage` via `/pw-next`, `/pw-setphase`.

**Manual update:** If the LLM completes a stage but didn't update:
```bash
/pw-setphase phasename=Shape  # preferred
/pw-setphase stage=2          # 0-indexed: 0=triage, 1=select, 2=setup, 3=context, etc.
```
