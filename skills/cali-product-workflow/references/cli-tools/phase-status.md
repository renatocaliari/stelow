# Phase Status Helper

> Utility to update workflow phase progress for human reviewers.

---

## Commands

| Command | Description |
|---------|-------------|
| `/pw:setphase phase=N` | Set current phase (0-indexed: 0=Setup, 1=Context, ...) |
| `/pw:next` | Advance to next phase |
| `/pw:status` | Show workflow status |

---

## Phase Index (0-indexed for CLI)

| Index | Phase Name | Skill Phase |
|-------|------------|-------------|
| 0 | Setup | Phase 1 |
| 1 | Context | Phase 2 |
| 2 | Shape | Phase 3 |
| 3 | Interface | Phase 4 |
| 4 | Critique | Phase 5 |
| 5 | Gate | Phase 6 |
| 6 | Scope Adjustment | Phase 7 |
| 7 | Interface Gate | Phase 8 |
| 8 | Interface Selection | Phase 9 |
| 9 | Tech Planning | Phase 10 |
| 10 | Execution | Phase 11 |
| 11 | Delivery Audit | Phase 12 |
| 12 | Complete | Workflow Done |

---

## Examples

```bash
# Show current status
/pw:status

# Set to Phase 3 (Shape Up)
/pw:setphase phase=2

# Advance to next phase
/pw:next
```

This is generated by the skill at the start of each phase.

---

## Auto-Update vs Manual

**Auto-update:** The extension updates `currentPhase` when user uses `/pw:next`, `/pw:setphase`.

**Manual update:** If the LLM completes a phase but the extension didn't update (e.g., LLM ran the skill standalone), the user can manually update:

```bash
/pw:setphase phase=N
```

Where N is the 0-indexed phase number (2 for Phase 3, 6 for Phase 7, etc.)

## Delivery Audit (Phase 12)

After all scopes are executed, the Delivery Audit phase verifies:

1. **Scope completeness** — all planned scopes implemented
2. **Gap analysis** — identify missing or extra features
3. **Quality gates** — tests passing, lint clean, build succeeds
4. **Lessons learned** — document what went well/what could improve

Outputs `delivery-audit.md` with findings and decision matrix.
