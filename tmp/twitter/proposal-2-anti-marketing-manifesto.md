# thread proposal 2 — anti-marketing / structure over vibes (v2)

## structure

1/ most product planning advice is either:
— "write a prd" (nobody reads it)
— "just start coding" (good luck)
— "use our ai tool" (still chaotic)

i wanted something between chaos and bureaucracy.

2/ stelow is an opinionated product workflow for ai agents and humans. stellar clarity. low friction.

it's an orchestrator — guides the process, keeps state across stages, coordinates tools without being tied to one environment. greenfield or brownfield, doesn't matter.

the stages use strategies from real product teams: shape up for scope, jtbd for user needs, adversarial critique for gaps, plannotator for visual gates. no theory — these are battle-tested approaches.

3/ the phases:

triage → shape (in/out, appetite, rabbit holes) → adversarial critique (fresh context reviewers) → plannotator gate → interface exploration (1-5 archetypes + hybrid, appetite-scaled) → tech planning (typed scopes with dependencies) → execution → verification.

appetite and mode control depth vs breadth. lean/auto = fast path. complete/full product = no shortcuts.

4/ the boring insight: constraints make better decisions. a mandatory gate before moving from shape to interface means you review before designing. typed scopes means dependencies are explicit before building.

saves rework. saves "we should have discussed that" conversations.

5/ i'm applying my background — years as developer, product manager, and consultant — into this workflow. it's not theory. i wrote about jtbd and experimentation before building this. now it's systematized inside an ai agent.

dogfooding every week. if you use ai coding agents and want sharper planning, give stelow a try. /sw-start and see how it feels.

feedback very welcome. no bs.
