# thread proposal 3 — the visual gate / plannotator (v2)

## structure

1/ the most important part of stelow isn't the stages. it's the gate.

after key phases (shape, interface), you run plannotator. it opens your plan as a visual document where you can comment point-by-point. approve or revise.

sound simple? it changes everything.

2/ before gates, my planning looked like: write some notes in the chat, feel good, start coding, realize i forgot something obvious, backtrack.

gates force a moment of honesty. the agent stops and says "let's review before moving on." you can't skip without looking at what you produced.

3/ stelow is an opinionated product workflow for ai agents and humans. stellar clarity. low friction.

it's an orchestrator — guides the workflow, keeps state between stages, uses tools agnostically. built for greenfield and brownfield projects.

the engine is a set of strategies from real product teams: shape up with appetite and in/out boundaries, jtbd, adversarial critique, interface exploration, visual gates. the orchestration makes them work together instead of isolated exercises.

appetite × mode controls depth: lean/core/complete for scope depth, auto/light/moderate/full product for how many gates and questions.

4/ example: during the critique phase, the agent asked me to list assumptions my shape was making. i had to stop and think. turned out two were wrong. caught before writing any code.

that's the value of structured gates inside an ai conversation.

5/ i'm the author and first user. my background — developer turned pm turned consultant — goes into every stage. i wrote about jtbd and experimentation before building this. now it's a workflow you can run with a slash command.

dogfooding every feature. when the gate catches my shortcuts, i know it works.

6/ if you use ai coding agents and struggle with planning, try stelow. free, open source, no signup. just /sw-start.

first tweet — no links. but if this resonates, reply or dm. happy to share the repo and hear your feedback.
