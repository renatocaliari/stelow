This directory defines the high-level concepts, business logic, and architecture of this project using markdown. It is managed by [lat.md](https://www.npmjs.com/package/lat.md) — a tool that anchors source code to these definitions.

Topics:
- [[architecture]] — System architecture, layers, stage numbering, CLI adapters
- [[business-rules]] — Core domain logic, workflow stages, Shape Up, Plannotator gate
- [[data-model]] — Entities, directory structure, file store types, data flow
- [[decisions]] — Architecture Decision Records (ADR-1 through ADR-5)
- [[glossary]] — Domain-specific terms and definitions
- [[guides]] — How-to guides for setup and development
- [[risks]] — Assumptions, open questions, and known risks
- [[tests]] — Test structure, test areas, and CI commands
